📖 HTML Structure


<!DOCTYPE html>

<html lang="en">

<head>

    <meta charset="UTF-8">

    <title>
        My Website
    </title>

</head>

<body>

</body>

</html>

📜 Notes

  • <!DOCTYPE html> tells the browser to use HTML5.
  • <html> is the root element of every webpage.
  • <head> contains metadata, links and page information.
  • <body> contains everything visible on the page.
  • Every HTML document should follow this basic structure.

📅 Added: 17 July 2026

✒️ Headings


<h1>Main Heading</h1>

<h2>Section Heading</h2>

<h3>Sub Heading</h3>

📜 Notes

  • HTML has six heading levels: h1 to h6.
  • h1 should usually be used once as the main page heading.
  • Use headings in order to create a logical page structure.
  • Headings improve accessibility and SEO.

📅 Added: 17 July 2026

✒️ Paragraphs


<p>
    This is a paragraph of text.
</p>

📜 Notes

  • Paragraphs are created with the <p> tag.
  • Browsers automatically add spacing between paragraphs.
  • Use CSS to customise spacing and appearance.

📅 Added: 17 July 2026

🔗 Links


<a href="https://example.com">
    Visit Website
</a>

📜 Notes

  • The href attribute defines the destination.
  • Use target="_blank" to open external links in a new tab.
  • Always include rel="noopener noreferrer" when using target="_blank".

📅 Added: 17 July 2026

🖼️ Images


<img 
src="image.jpg" 
alt="Description of image">

📜 Notes

  • The src attribute specifies the image location.
  • The alt attribute describes the image.
  • Always include meaningful alt text for accessibility.
  • Use CSS to control image size and positioning.

📅 Added: 17 July 2026

📜 Lists


<ul>

    <li>HTML</li>
    <li>CSS</li>
    <li>JavaScript</li>

</ul>


<ol>

    <li>Step One</li>
    <li>Step Two</li>

</ol>

📜 Notes

  • <ul> creates a bulleted list.
  • <ol> creates a numbered list.
  • <li> defines each list item.
  • Lists improve readability and organisation.

📅 Added: 17 July 2026

🏛️ Semantic Structure


<header>

<main>

<section>

<article>

<footer>

📜 Notes

  • Semantic elements describe the purpose of their content.
  • They improve accessibility and SEO.
  • Use semantic elements instead of unnecessary <div> tags where possible.
  • Common semantic elements include <header>, <main>, <section>, <article> and <footer>.

📅 Added: 17 July 2026

🧙 Forms


<form>

<label>
Username:
</label>

<input 
type="text"
name="username">

<button>
Submit
</button>

</form>

📜 Notes

  • Forms collect information from users.
  • <label> improves accessibility.
  • <input> defines the type of user input.
  • <button> is commonly used to submit forms.

📅 Added: 17 July 2026

🧱 Div Containers


<div class="card">

<h2>
Title
</h2>

<p>
Content
</p>

</div>

📜 Notes

  • <div> is a generic container element.
  • It is commonly used for layouts and grouping content.
  • Divs have no semantic meaning.
  • Prefer semantic HTML elements where appropriate.

📅 Added: 17 July 2026

🏷️ Classes and IDs


<div class="card" id="main-card">

</div>

📜 Notes

  • class="" → reusable.
  • id="" → unique.
  • CSS class → .card
  • CSS ID → #main-card
  • JS class → .card
  • JS ID → #main-card

📅 Added: 17 July 2026

⚡ JavaScript + CSS Link


<script src="javascript.js" defer></script>
<link rel="stylesheet" href="style.css">

📜 Notes

  • Defer waits until HTML has loaded before executing JavaScript.
  • Prevents querySelector() from returning null because the elements haven't been created yet.
  • Ideally both placed at the top in the head.

📅 Added: 17 July 2026

🔘 Buttons


<button>
    Click Me
</button>

📜 Notes

  • Creates a clickable button.
  • Can be used inside or outside forms.
  • Usually controlled with JavaScript using addEventListener().
  • Common types:
    type="button" → normal button.
    type="submit" → submits a form.
    type="reset" → resets form inputs.

📅 Added: 17 July 2026

💬 HTML Comments


< !-- This is a comment -- >

📜 Notes

  • Comments are ignored by the browser. Remove spaces after "<>"
  • Used to explain code or leave reminders.
  • Shortcut in VS Code:
    Windows: Ctrl + /
    Mac: Cmd + /

📅 Added: 17 July 2026

📁 File Paths


<img src="./images/photo.png">

📜 Notes

  • ./ → current folder.
  • ../ → go back one folder.
  • Absolute path → full URL/location.
  • Relative paths are usually preferred in projects.

📅 Added: 17 July 2026

🌐 Favicon


<link rel="icon" href="favicon.ico">

📜 Notes

  • Adds the small icon shown in the browser tab.
  • Goes inside <head>.
  • Common image types:
    .png
    .svg
    .ico
  • Relative paths are usually preferred in projects.

📅 Added: 17 July 2026

📊 Tables


<table>

<tr>
<th>Name</th>
<th>Level</th>
</tr>

<tr>
<td>HTML</td>
<td>Beginner</td>
</tr>

</table>

📜 Notes

  • <table>creates the table.
  • <tr>creates a row.
  • <th>creates a heading cell.
  • <td>creates a normal cell.
  • Don't use tables for layouts.

📅 Added: 17 July 2026

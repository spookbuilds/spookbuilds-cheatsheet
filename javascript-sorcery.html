⚡ Variables (let & const)


// Can be changed

let playerName = "RJ";


// Cannot be changed

const gameVersion = "1.0";

📜 Notes

  • Variables store information so JavaScript can remember and reuse it.
  • let creates a variable that can be reassigned.
  • const creates a variable that cannot be reassigned.
  • Modern JavaScript uses const by default unless the value needs to change.
  • Variable names should explain what the data represents.
  • JavaScript naming convention uses camelCase: playerScore, userName

🧩 Structure

  • const / let → creates the variable.
  • playerName → variable name.
  • = → assigns a value.
  • "RJ" → stored data.

⚡ Quick Reference

Keyword Can Change?
const ❌ No
let ✅ Yes
var ⚠️ Old (avoid)

🔗 Often Used With

  • Functions → storing results and values.
  • Arrays → storing collections of data.
  • Objects → storing grouped information.
  • DOM → storing HTML elements.

🔄 Event Loop

    console.log("Timer");

📜 Notes

  • JavaScript runs one task at a time.
  • The event loop manages tasks waiting to run.
  • Async tasks are moved away until they are ready.
  • The event loop checks when queued tasks can execute.
  • This allows JavaScript to handle timers, clicks and APIs without freezing.

🧩 Structure

  • Call Stack → current code running.
  • Web APIs → browser handles timers/events.
  • Callback Queue → waiting tasks.
  • Event Loop → moves tasks back when stack is empty.
